다익스트라(Dijkstra) 알고리즘

여러 코딩테스트를 보면서 생각했던 건 확실히 기본기가 중요하다는 것이다. 그래서 기본기중 기본기들을 공부하기로 마음먹었다. 먼저 다익스트라 알고리즘이다.

다익스트라

다익스트라 알고리즘은 한 노드로부터 모든 다른 노느까지의 최단거리를 구할 수 있는 알고리즘이다.

그러나 최단거리는 BFS도 있다. BFS와 뭐가 다를까? 바로 bfs는 노드가 없는 경우도 있고 가중치가 없는 경우도 있다. 숨바꼭질 문제 같은 경우 2 → 4 → 8 등 위치가 바뀌었다고 해야한다. 가중치가 아니다. 즉, 가중치가 존재하면 다익스트라 알고리즘으로 풀어내는 것이 일반적이다.

특징

  • 한 노드에서 모든 노드까지의 최단거리를 구할 수 있다.
  • 단, 음수 가중치가 존재해선 안된다.
  • 사이클이 존재해선 안된다.
  • 방향 그래프를 기반으로 한다.

이론

  • BFS가 베이스이다.
  • 가장 가까운 노드를 선택한다.

변수

  • dist[A] = B 시작 노드가 S 라고할 때, B = S 노드와 A노드의 최소 거리
  • check[A]. 각각의 노드를 방문하였는지 검사

구현

  • 간선이 주어지지 않은 곳은 INF 처리
  • 가장 가까운 노드부터 방문하며 현재 dist 배열과 비교하며 값을 갱신한다.

그림으로 이해하기

이와 같은 간단한 그래프가 있다고 가정하고 1번 노드에서 각 노드까지의 비용을 다익스트라 알고리즘을 적용해서 구해보자.

여기서 dist[1] = INF

dist[2] = 4 처럼 이루어진다.

정점 번호1234
거리INF4INF10
방문표시truefalsefalsefalse

처음은 이렇게 된다. 1번은 자기 자신이기 때문에 INF 혹은 0으로 정해주면 된다. 그리고 3번까지 한번에 가는 간선이 없기 때문에 지금은 INF이다.

먼저 가장 비용이 적은 2번 노드부터 방문한다. 그리고 2번의 인접노드들을 탐색한다. 2번을 방문했을 때 표는 다음과 같다. 이전 표와 다른 점이 없지만 2번노드에 방문표시가 된다.

정점 번호1234
거리INF4INF10
방문표시truetruefalsefalse

4번이 인접노드이고 아직 방문하지 않았으므로 방문한다.

4번을 방문하고서 체크해본다. dist[4] 즉 지금 1번부터 4번까지의 비용과, dist[2]에서 4로 넘어오는 비용을 더했을 때 뭐가 더 싼가? 즉 바로 다이렉트로 왔을 때와 2번을 거쳐서 왔을 때 무엇이 더 싼가를 검토한다.

dist[4] = 10이고

dist[2] + 1 = 5이다. 그러므로 2번을 거쳐오는게 싸다! 그러므로 갱신해준다.

정점 번호1234
거리INF4INF5
방문표시truetruefalsetrue

이제 4번의 간선을 찾아보자. 3번과 연결되어있다. 3번으로 이동한다.

3번은 INF이다. 1번에서 연결됐던 적이 없기 때문이다. 그러므로 당연히 갱신이 된다. dist[4] + 6 = 11

정점 번호1234
거리INF4115
방문표시truetruetruetrue

위 표가 1번에서 가장 싼 경우의 수들이다.

그러므로 갱신 조건을 코드로 표현하면

if(dist[next] > dist[now] + next_value ){
		dist[next] = dist[now] + next_value;
		pq.push(next, next_value);
}
